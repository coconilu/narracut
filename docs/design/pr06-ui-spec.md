# PR06 工作台应用壳设计基准

## 目标

PR06 只建立“项目首页 + 桌面工作台应用壳”，不提前实现 PR07 的完整阶段编辑器。
界面必须让用户一眼看出 NarraCut 的核心对象是阶段、运行、产物和审核，不是万能聊天框。

由于内置 Image Gen 连续返回网络错误，用户已明确同意使用 HTML/CSS 静态概念截图作为
本 PR 的视觉基准。概念源文件为 `pr06-workbench-concept.html`。

## 信息架构

### 项目首页

- 顶部品牌、数据位置说明和设置入口；
- 最近项目使用行式列表，不使用卡片网格；
- 行内展示当前阶段、阶段状态、最后打开时间和路径可用性；
- 主操作为“新建项目”和“打开项目”；
- 新建项目使用右侧抽屉，字段为项目名称、父目录、目录名、工作流和语言；
- 归档项目默认隐藏，可显式显示；
- 路径不可用时保留索引记录并提供清晰状态，不伪造可打开能力。

### 工作台

- 顶栏：返回项目、项目名、当前阶段、执行器、运行、停止、历史、导出；
- 左栏：9 个固定阶段与叙事进度线；
- 中央：主编辑/预览画布；PR06 只提供结构化占位内容和视图切换；
- 右栏：执行器、模型、输入依赖和配置快照摘要；
- 底栏：事件、日志、产物三个页签和任务进度/错误恢复入口；
- 不提供聊天入口，不允许 UI 拼接 shell、FFmpeg 或 Codex 参数。

## 视觉系统

| Token | 值 | 用途 |
|---|---|---|
| `--ink-950` | `#111415` | 窗口底色 |
| `--ink-900` | `#171a1b` | 顶栏与侧栏 |
| `--ink-850` | `#1d2122` | 次级面板 |
| `--ink-800` | `#252a2b` | hover/选中底色 |
| `--line` | `#303637` | 1px 分隔线 |
| `--paper` | `#f5f2eb` | 文档画布 |
| `--paper-line` | `#ded9cf` | 文档边界 |
| `--text` | `#ece9e2` | 深色界面主文字 |
| `--muted` | `#929a98` | 辅助文字 |
| `--accent` | `#e36043` | 运行、当前阶段、主操作 |
| `--approved` | `#76a37d` | 已采用 |
| `--stale` | `#d3a04e` | 已过期/警告 |
| `--danger` | `#d97868` | 失败/停止 |

- UI 字体：`Segoe UI`、`Microsoft YaHei UI`、系统无衬线；
- 文档标题：`Songti SC` / `SimSun`，只用于中央文档；
- 控件圆角 6px，面板圆角最多 10px；
- 阴影只用于文档纸张和抽屉，不使用玻璃、渐变和霓虹光；
- 图标使用一致的 1.7px 线性 SVG，状态点使用实心圆；
- 动效 140–180ms，只用于 hover、抽屉和面板切换；遵守 `prefers-reduced-motion`。

## 组件与职责

| 组件 | 职责 |
|---|---|
| `AppShell` | 在项目首页与工作台之间组合，不承载业务细节 |
| `ProjectHome` | 最近项目、归档过滤、创建/打开入口和错误状态 |
| `CreateProjectDrawer` | 有类型的新建项目表单 |
| `WorkbenchShell` | 顶栏、阶段导航、画布、检查器和事件面板布局 |
| `StageRail` | 阶段状态投影与当前阶段选择 |
| `PreviewCanvas` | PR06 的只读阶段概览与预览模式切换 |
| `InspectorPanel` | 当前阶段配置摘要，不直接写原始配置文件 |
| `ActivityPanel` | Job/索引摘要、事件列表与恢复入口 |
| `StatusGlyph` | 统一状态颜色、图标与无障碍文本 |

## 固定可见文案

首屏允许出现的主要文案：

- 品牌：`NarraCut`、`叙剪`；
- 首页：`项目`、`新建项目`、`打开项目`、`最近使用`、`显示已归档`；
- 工作台操作：`运行`、`停止`、`历史`、`导出`；
- 工作台页签：`预览`、`结构`、`事件`、`日志`、`产物`；
- 9 个阶段：`创作简报`、`事实研究`、`事实脚本`、`口播音频`、`字幕`、`场景规划`、
  `时间轴`、`渲染`、`导出`。

不新增营销口号、KPI、装饰徽章、聊天提示或未实现功能入口。

## 响应式边界

- 设计基准视口：1600 × 1000；
- 1280px 以下：右侧检查器折叠为抽屉，阶段栏缩为 184px；
- 900px 以下：工作台进入紧凑模式，阶段栏和检查器均由工具栏打开；
- 最小支持宽度 720px；项目列表允许横向滚动，不改造成卡片网格。

## 核心交互

1. 首页加载最近项目；失败时显示结构化错误和重试；
2. 选择最近项目后调用 typed `open_project`，先完成崩溃恢复，再并行读取一致的 workflow 与 jobs 投影；
3. 新建项目成功后初始化 workflow 并进入工作台；
4. 阶段切换只更新当前视图，不伪造阶段运行；
5. “运行”在 PR06 仅对已具备受控输入的阶段开放；不可运行时说明原因；
6. 任务面板可以调用 `recover_jobs` 并呈现索引状态；停止操作只绑定当前阶段唯一活动任务并附带原因；
7. 浏览器开发模式使用明确的本地演示适配器，Tauri 环境始终调用真实 typed commands。
