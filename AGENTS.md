# AGENTS.md

请使用中文沟通，并维护中文优先的用户文档。

## 产品边界

- NarraCut 是“可观察、可编辑、可重跑”的视频创作工作台，不是一次性成片生成器。
- 项目、阶段运行、配置快照、中间产物和最终导出都必须可追踪。
- 任何局部重生成都应明确影响范围，不得静默覆盖历史运行。
- 远程 API 与本地 Codex CLI 必须位于统一的 AI Provider 接口之后。
- HTML、FFmpeg、HyperFrames 等实现必须位于 Renderer 接口之后。

## 架构约束

- 前端只通过有类型的 Tauri commands 使用本地能力，不直接拼接或执行 shell 命令。
- 长任务异步返回 `job_id`，必须支持取消、重试、进度报告和幂等执行。
- 阶段输出应包含输入引用、配置快照、产物清单、日志摘要和状态。
- 音频与视频阶段只消费已经审核通过的结构化契约，不从原始资料重新自由总结。
- 事实性脚本、字幕和场景应保留 `claim_id` 与 `evidence_ref` 的追溯关系。
- MCP 或 Agent 工具只暴露高层、有界操作，不开放任意 shell、FFmpeg 参数或插件安装。

## 工程约束

- 跨 Rust、TypeScript 或外部服务的行为，先定义版本化契约，再实现适配器。
- React 组件保持小而聚焦；`App` 只负责组合，不承载完整编辑器逻辑。
- 新依赖要记录用途、许可证和可替代方案。
- 测试优先覆盖项目迁移、引用链、任务重试、局部重生成和最终 manifest。
- 不要向 Git 提交模型、原始资料、密钥、缓存或生成媒体；授权明确的小型测试夹具除外。

## 合规约束

- 声音克隆与人物肖像必须关联明确授权记录。
- 外部素材必须保存来源、作者、许可证、署名文本和内容哈希。
- 代码许可证不能替代模型、字体、权重或内容许可证。
- 生成式图片与视频只能作为表达素材，不能作为事实证据。
